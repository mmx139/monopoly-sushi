# 复盘：Bug状态误改

## 问题描述
修复bug后直接修改状态为"Fixed"，违反状态定义规范。

## 时间线
| 时间 | 事件 |
|---|---|
| 多次 | 修复B00002、B00004、B00012等bug后，直接把状态改为Fixed |
| 发现 | 用户指出状态被错误标记，只有用户可以改为Fixed |
| 纠正 | 将Open状态改为Pending on Verify |
| 教训 | 需要将规范提升到全局设定 |

## 根因
- **表面原因**：没有仔细阅读README.md中的状态定义
- **真正原因**：对规范理解不深刻，习惯性按旧方式操作

## 解决方案
1. 将bug状态纠正回Pending on Verify
2. 更新README.md，明确"只有用户可以改为Fixed"
3. 将Bug追踪规范提升到全局CLAUDE.md

## 教训
1. 修复bug后只更新修复时间和根因，不改变状态
2. 状态流转：Open → Pending on Verify → Fixed（只有用户能改Fixed）
3. 新规范需要及时同步到全局CLAUDE.md

## 预防措施
- 新规范发布后，及时同步到全局CLAUDE.md
- 遵守状态定义，不越权操作
